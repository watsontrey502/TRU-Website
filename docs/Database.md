# Database

All data lives in **Supabase** (hosted Postgres). Schema is in `supabase/schema.sql`.

## Tables

| age | text | |
| gender | text | |
| instagram | text | |
| neighborhood | text | |
| work | text | |
| heard_from | text | How they found TRU |
| interesting | text | "What makes you interesting?" |

**Access**: Only admins can read this table. The API route inserts using the service role.

### double_take_votes
After an event, each person picks who they liked. Mutual votes = match.

| Column | Type | Notes |
|--------|------|-------|
| event_id | uuid | |
| voter_id | uuid | Who voted |
| voted_for_id | uuid | Who they voted for |

---

### event_round_groups
Generated by the grouping algorithm. Assigns people to numbered groups for each phase of the event.

| Column | Type | Notes |
|--------|------|-------|
| event_id | uuid | |
| phase_index | integer | Which phase of the event |
| group_number | integer | Group 1, 2, 3, etc. |
| profile_id | uuid | Who's in the group |

## Running the Schema
1. Go to your Supabase project dashboard
2. Click **SQL Editor**
3. Paste the contents of `supabase/schema.sql`
4. Click **Run**

This creates all tables, enums, RLS policies, functions, and seeds sample events.
